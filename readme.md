# Nouxis Protocol — IDL

Interface Definition Language (IDL) files for the **Nouxis Protocol** Solana program.

```
Program ID: NouXXXZsWXpanM5UzshMKZH4wUbeFNcxPWnFyTBgRP1
```

## Files

| File       | Description                                                           |
| ---------- | --------------------------------------------------------------------- |
| `Nouxis.json` | Anchor IDL (JSON) — used by clients, explorers, and tooling           |
| `Nouxis.ts`   | TypeScript types — generated by Anchor, used with `@coral-xyz/anchor` |

## Usage

### With `@coral-xyz/anchor`

```typescript
import { Program, AnchorProvider } from "@coral-xyz/anchor";
import { Connection, PublicKey } from "@solana/web3.js";
import type { Nouxis } from "./Nouxis";
import idl from "./Nouxis.json";

const PROGRAM_ID = new PublicKey("NouXXXZsWXpanM5UzshMKZH4wUbeFNcxPWnFyTBgRP1");
const connection = new Connection("https://api.devnet.solana.com");
const provider = new AnchorProvider(connection, wallet, {});
const program = new Program<Nouxis>(idl as any, provider);
```

### Fetch an account

```typescript
// Protocol config PDA
const [configPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("config")],
  PROGRAM_ID,
);
const config = await program.account.protocolConfig.fetch(configPda);

// Agent identity PDA
const [agentPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("agent"), nftMint.toBuffer()],
  PROGRAM_ID,
);
const agent = await program.account.agentIdentity.fetch(agentPda);
```

### Call an instruction

```typescript
await program.methods
  .registerAgent(metadataUri, metadataHash, feedbackAuthority, tags)
  .accounts({
    config: configPda,
    agentIdentity: agentPda,
    nftMint: nftMint,
    collection: collectionPubkey,
    owner: wallet.publicKey,
    feeTreasury: treasuryPubkey,
    mplCoreProgram: MPL_CORE_ID,
    systemProgram: SystemProgram.programId,
  })
  .rpc();
```

### Decode with `@solana/web3.js` only (no Anchor)

```typescript
import idl from "./Nouxis.json";
import { BorshAccountsCoder } from "@coral-xyz/anchor";

const coder = new BorshAccountsCoder(idl as any);
const accountInfo = await connection.getAccountInfo(agentPda);
const decoded = coder.decode("agentIdentity", accountInfo.data);
```

## Program Architecture

### Modules & Instructions (23 total)

| Module         | Instructions                                                                                                 | Description                                              |
| -------------- | ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------- |
| **Admin**      | `initialize`, `update_config`                                                                                | Protocol setup and config management                     |
| **Identity**   | `register_agent`, `claim_agent`, `deactivate_agent`, `reactivate_agent`                                      | Agent NFT lifecycle                                      |
| **Service**    | `add_service`, `remove_service`, `update_service`                                                            | Service endpoint registration (MCP, A2A, API, Web)       |
| **Reputation** | `give_feedback`, `revoke_feedback`                                                                           | Feedback scoring (0-100, ERC-8004 aligned)               |
| **Validation** | `validation_request`, `validation_response`                                                                  | Two-step validator attestation                           |
| **Delegation** | `create_delegation`, `revoke_delegation`, `fund_delegation`, `withdraw_delegation`, `set_feedback_authority` | Granular permission delegation (max 3 levels)            |
| **Payment**    | `set_payment_requirement`, `update_payment_requirement`, `verify_and_settle`                                 | x402 payment settlement (Atomic, Attestation, Delegated) |
| **Vault**      | `close_receipt`                                                                                              | Payment receipt cleanup                                  |

### Account Types (9 PDAs)

| Account              | Seed Pattern                               | Description                                       |
| -------------------- | ------------------------------------------ | ------------------------------------------------- |
| `ProtocolConfig`     | `["config"]`                               | Global protocol settings, fees, authority         |
| `AgentIdentity`      | `["agent", nft_mint]`                      | Agent metadata, reputation scores, owner tracking |
| `AgentService`       | `["service", nft_mint, service_type]`      | Registered service endpoints                      |
| `Delegation`         | `["delegation", nft_mint, delegation_id]`  | Permission grants with spend limits and expiry    |
| `FeedbackEntry`      | `["feedback", nft_mint, reviewer]`         | Score 0-100 with optional payment proof           |
| `ValidationRequest`  | `["validation_req", nft_mint, validator]`  | Pending validation requests                       |
| `ValidationResponse` | `["validation_resp", nft_mint, validator]` | Validator attestation results                     |
| `PaymentRequirement` | `["payment_req", nft_mint, service_type]`  | Pricing config per service                        |
| `PaymentReceipt`     | `["receipt", payment_id]`                  | Settlement proof with fee breakdown               |

### Events (19)

The program emits events for all state changes: `AgentRegistered`, `AgentClaimed`, `PaymentSettled`, `DelegationCreated`, `FeedbackGiven`, `ServiceAdded`, and more. Subscribe via Anchor's `addEventListener` or parse from transaction logs.

## Key Design Notes

- **agent_id = NFT mint pubkey** — no counters or hashes
- **Owner follows NFT** — transferring the NFT transfers the agent and its revenue stream
- **owner_version** — incremented on `claim_agent`, invalidates all stale delegations
- **Settlement fee** — 300 bps (3%) protocol fee, configurable up to 1000 bps (10%)
- **Metaplex Core** NFTs — all agents belong to a single Core Collection
